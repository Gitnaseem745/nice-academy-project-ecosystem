# Learn in Public Workflow

This workflow is **mandatory** for every project in the ecosystem.

Learning is considered incomplete if this workflow is skipped.

---

## Stage 1 — Start Announcement

Before building, share:
- What you are building
- Why you chose it
- What you want to learn

Platforms:
- LinkedIn (mandatory)
- Twitter / X (optional)

---

## Stage 2 — Progress Updates

During building, share:
- Small wins
- Problems faced
- Key learnings

Rules:
- Post every 2–3 days
- Keep posts honest and short
- Do not teach or tutorialize

---

## Stage 3 — Project Launch

After completion, share:
- What you built
- Tools used
- What you learned
- Project links

This is your **public proof of work**.

---

## Approved Platforms by Role

- Web Developer → LinkedIn, GitHub
- Web Designer → LinkedIn, Dribbble, Behance
- Graphic Designer → LinkedIn, Behance

---

## What NOT to Do

- Do not copy captions
- Do not pretend expertise
- Do not post solution code
- Do not skip stages

---

## Why This Matters

Public learning:
- Builds visibility
- Creates accountability
- Compounds over time

**Rules:**

* Start with `/guide`
* Do NOT open `/solution` early
* Rebuild this project in your own repository
* Share progress publicly

---

## Project Structure

```text
project-name/
├── README.md
├── project-intent.md
├── guide/
│   ├── en/
│   └── hi-en/
├── solution/
├── reference/
└── reflection.md
```

---

## Language Options

* English (Canonical): `guide/en/`
* Hinglish (Hindi + English): `guide/hi-en/`

Choose **one language** and follow it completely.

---

## Completion Checklist

* [ ] Followed all guide steps
* [ ] Built the project independently
* [ ] Used solution only if stuck
* [ ] Rebuilt project in personal repo
* [ ] Shared publicly (Learn in Public)
* [ ] Completed reflection
